10 · Scheduler Concept

Standard section: [exec.sched] stdexec reference: include/stdexec/concepts.hpp Related tests: tests/test_thread_pool.cpp · tests/test_custom_scheduler.cpp, exec/inline_scheduler.hpp

Definition

Standard definition (exec.tex:310):

A scheduler is an abstraction of an execution resource with a uniform, generic interface for scheduling work onto that resource.

cpp
namespace std::execution {
  struct scheduler_tag {};

  template<class Sch>
  concept scheduler =
    std::destructible<Sch> &&
    std::copy_constructible<Sch> &&
    std::equality_comparable<Sch> &&
    requires(Sch&& sch, const Sch& csch) {
      { schedule(static_cast<Sch&&>(sch)) } -> sender;
      { csch.query(get_forward_progress_guarantee) }
        -> same_as<forward_progress_guarantee>;
    };
}

A scheduler must satisfy:

  1. Destructible, copy constructible, equality comparable
  2. schedule(sch) returns a sender (which executes on that execution resource)
  3. query(get_forward_progress_guarantee) returns the concurrency guarantee

forward_progress_guarantee

cpp
enum class forward_progress_guarantee {
  concurrent,       // may execute concurrently, but may block each other
  parallel,         // true parallel execution (multi-core)
  weakly_parallel   // weak parallelism (e.g. GPU warp)
};

Guarantees for different schedulers:

SchedulerGuarantee
inline_schedulerconcurrent (on the current thread)
static_thread_poolparallel (multi-threaded)
nvexec::stream_schedulerweakly_parallel (GPU)

Behavior of schedule

schedule(sch) returns a sender whose asynchronous operation will:

  1. Acquire an execution agent on the execution resource associated with sch
  2. Execute set_value(rcvr) (no arguments) on that execution agent
  3. Upon completion, the value completion scheduler is sch itself
cpp
scheduler auto sch = pool.get_scheduler();

// equivalent to "do nothing on pool's thread"
sender auto s = schedule(sch);

// practical form: execute work on the thread pool
sender auto work = schedule(sch)
                 | then([] { /* execute on thread pool thread */ });

Built-in Scheduler Comparison

inline_scheduler

cpp
class inline_scheduler {
  // the sender returned by schedule(sch) executes immediately on the current thread
  // essentially completes synchronously on the start() call stack
};

Implementation essence: inline_scheduler::schedule() returns a sender whose start() directly calls set_value(rcvr).

static_thread_pool::scheduler

cpp
exec::static_thread_pool pool{4};
auto sch = pool.get_scheduler();

// the sender returned by schedule(sch) will:
// 1. submit the work to the thread pool's queue
// 2. a thread in the pool picks up the work
// 3. calls set_value(rcvr) on that thread

run_loop::scheduler

cpp
stdexec::run_loop loop;
auto sch = loop.get_scheduler();

// start a thread to process events
std::thread t([&loop] { loop.run(); });

// schedule(sch) submits work to the run_loop's queue
// loop.run() consumes that work

How to Write a Scheduler

cpp
// Corresponding tests: test_custom_scheduler.cpp, test_cpo.cpp → "custom scheduler with member schedule() is callable"
struct my_scheduler {
  // concurrency guarantee
  friend auto tag_invoke(
    stdexec::tag_t<stdexec::get_forward_progress_guarantee>,
    const my_scheduler&) noexcept -> stdexec::forward_progress_guarantee {
    return stdexec::forward_progress_guarantee::concurrent;
  }

  // equality comparison
  bool operator==(const my_scheduler&) const = default;
};

// custom sender returned by schedule
struct my_schedule_sender {
  using sender_concept = stdexec::sender_tag;

  template<stdexec::receiver R>
  struct operation {
    R rcvr_;
    void start() & noexcept {
      // execute on the actual execution resource
      stdexec::set_value(static_cast<R&&>(rcvr_));
    }
  };

  template<stdexec::receiver R>
  auto connect(R r) && -> operation<R> {
    return {std::move(r)};
  }
};

// implement the schedule CPO via tag_invoke
my_schedule_sender tag_invoke(
  stdexec::tag_t<stdexec::schedule>, const my_scheduler&) {
  return {};
}

Scheduler Responsibility Boundaries

A scheduler is not responsible for:

  • Defining how to switch execution contexts (that is the responsibility of starts_on/continues_on)
  • Managing the lifecycle of operation states (that is the responsibility of operation_state)
  • Determining how completion signals propagate (that is the responsibility of sender adaptors)

A scheduler does one thing: provides a schedule sender that guarantees calling set_value on its resource.

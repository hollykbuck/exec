12 · Completion Signatures

Standard section: [exec.cmplsig] [exec.getcomplsigs] stdexec reference: include/stdexec/__detail/__basic_sender.hpp Related tests: tests/test_completion_signatures.cpp

Definition

Completion signatures are the compile-time description of all possible completion operations a sender can produce.

Standard definition (exec.tex:354):

A completion signature is a function type that describes a completion operation.

cpp
template<class Fn>
concept completion-signature = // function type
  std::is_function_v<Fn>;

// for example:
set_value_t(int, double)           // value completion signature
set_error_t(std::exception_ptr)    // error completion signature
set_stopped_t()                     // stopped completion signature

completion_signatures Class Template

cpp
namespace std::execution {
  template<completion-signature... Fns>
  struct completion_signatures {};
}

Usage:

cpp
// Corresponding test: test_completion_signatures.cpp → "completion_signatures with multiple signatures"
// just(42) completion signatures:
// success: set_value(42)
// failure: never occurs
// cancellation: never occurs
using just_42_sigs =
  completion_signatures<set_value_t(int)>;

// a potentially-failing sender:
using maybe_fail_sigs =
  completion_signatures<
    set_value_t(int),
    set_error_t(std::exception_ptr)
  >;

// a cancellable sender:
using cancellable_sigs =
  completion_signatures<
    set_value_t(int),
    set_error_t(std::exception_ptr),
    set_stopped_t()
  >;

Querying Completion Signatures

cpp
template<class Sndr, class... Env>
  requires sender_in<Sndr, Env...>
using completion_signatures_of_t =
  decltype(get_completion_signatures<Sndr, Env...>());

// usage
using Sigs = completion_signatures_of_t<just<42>>;
// Sigs = completion_signatures<set_value_t(int)>

Examples:

cpp
// Corresponding test: test_completion_signatures.cpp → "value_types_of_t extracts value types"
using Just = decltype(just(42, 3.14));

// value_types_of_t<Just> = variant<tuple<int, double>>
// error_types_of_t<Just> = variant<>
// sends_stopped<Just>    = false

using Failable = decltype(just_error(std::exception_ptr{}) | upon_error([](auto) { return 0; }));
// value_types_of_t<Failable> = variant<tuple<int>>
// error_types_of_t<Failable> = variant<>  (upon_error consumed the error)

gather-signatures Utility

The standard defines an exposition-only utility gather-signatures for processing completion signatures:

cpp
// collect all function types matching a given Tag from completion_signatures<Sigs...>
template<auto Tag, class Sigs,
         template<class...> class Tuple,
         template<class...> class Variant>
using gather-signatures = /* recursive expansion */;

// for example: from completion_signatures<set_value_t(int), set_value_t(double), set_error_t(ptr)>
// gather set_value signatures:
// gather-signatures<set_value_t, Sigs, decayed-tuple, variant>
// → variant<tuple<int>, tuple<double>>

Declaring Completion Signatures for Custom Senders

Senders can declare completion signatures in two ways:

Way 1: Static Member Type

cpp
// Corresponding test: test_completion_signatures.cpp → "completion_signatures_of_t for just"
struct my_sender {
  using sender_concept = stdexec::sender_tag;
  // fixed completion signatures
  using completion_signatures =
    stdexec::completion_signatures<
      stdexec::set_value_t(int),
      stdexec::set_error_t(std::exception_ptr)
    >;
};

Way 2: get_completion_signatures Member Function Template

When completion signatures depend on the environment or the sender’s value category:

cpp
// Corresponding test: test_impl_then.cpp
struct env_dependent_sender {
  using sender_concept = stdexec::sender_tag;

  template <class Self, class... Env>
  static consteval auto get_completion_signatures()
    -> stdexec::completion_signatures</* ... */>
  {
    return {};
  }
};

This is the pattern used by the retry algorithm (examples/algorithms/retry.hpp):

cpp
template <class Self, class... Env>
static consteval auto get_completion_signatures()
  -> stdexec::__transform_completion_signatures_t<
    stdexec::completion_signatures_of_t<S&, Env...>,
    stdexec::completion_signatures<stdexec::set_error_t(std::exception_ptr)>,
    _value,
    _error>
{
  return {};
}

__transform_completion_signatures_t

stdexec provides a utility for transforming completion signatures:

cpp
template<class Sigs,                    // input completion signatures
         class Default = completion_signatures<>,      // default signatures to add
         template<class...> class ValueFn,  // value signature transformation
         template<class...> class ErrorFn,  // error signature transformation
         bool SendStopped = /* ... */>      // whether to pass through stopped
using __transform_completion_signatures_t = /* ... */;

This is very useful when writing sender adaptors — deriving the parent sender’s completion signatures from the child sender’s completion signatures.

Role of Completion Signatures

  1. Compile-time validation: connect() checks receiver_of<R, Sigs>
  2. Type extraction: sync_wait deduces the return type via value_types_of_t
  3. Algorithm adaptation: when_all computes its own completion signatures from all child senders’ completion signatures
  4. Domain transformation: transform_sender decides algorithm implementation based on completion signatures

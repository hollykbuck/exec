13 · Sender Factory: just / just_error / just_stopped

Standard section: [exec.just] stdexec reference: include/stdexec/__detail/__just.hpp Related tests: tests/test_basics.cpp

Overview

Sender factories create senders from non-sender values. The simplest factories are the just family:

cpp
namespace std::execution {
  inline constexpr unspecified just = /* ... */;
  inline constexpr unspecified just_error = /* ... */;
  inline constexpr unspecified just_stopped = /* ... */;
}

just(args...) — Immediate Success

cpp
// Corresponding test: test_basics.cpp → "just creates an immediate-value sender"
// Simplest sender: immediately sends set_value()
sender auto s1 = just();
// completion_signatures = <set_value_t()>

// With a value
sender auto s2 = just(42);
// completion_signatures = <set_value_t(int)>

// With multiple values
sender auto s3 = just(1, 2.0, "hello");
// completion_signatures = <set_value_t(int, double, const char*)>

Implementation

The sender returned by just(v) immediately synchronously calls set_value(rcvr, v) in start():

cpp
// Corresponding test: test_basics.cpp
// pseudocode
void start() & noexcept {
  stdexec::set_value(std::move(rcvr_), args_...);
}

Use Cases

cpp
// Corresponding test: test_basics.cpp → "empty just sends void"
// As a pipeline starting point
sync_wait(
  just(42)
    | then([](int x) { return x * 2; })
);
// Result: 84

// Returning a fixed value in let_value
sender auto handler = let_value(
  some_sender,
  []() { return just(0); }  // return fixed value on success
);

just_error(err) — Immediate Failure

cpp
// Corresponding test: test_basics.cpp → "just_error sends an error"
// Immediately sends set_error()
sender auto e = just_error(std::make_exception_ptr(std::runtime_error{"oops"}));
// completion_signatures = <set_error_t(std::exception_ptr)>

// Also supports error_code
std::error_code ec = std::make_error_code(std::errc::io_error);
sender auto e2 = just_error(ec);

Use Cases

cpp
// Corresponding test: test_basics.cpp
// Return an error in a conditional branch
sender auto validate(int x) -> sender auto {
  if (x < 0)
    return just_error(std::make_exception_ptr(std::invalid_argument{"negative"}));
  return just(x);
}

just_stopped() — Immediate Cancellation

cpp
// Corresponding test: test_basics.cpp → "just_stopped sends a stopped signal"
// Immediately sends set_stopped()
sender auto s = just_stopped();
// completion_signatures = <set_stopped_t()>

Use Cases

cpp
// Cancel in a conditional branch
sender auto maybe_cancel(bool should_stop) -> sender auto {
  if (should_stop)
    return just_stopped();
  return just(42);
}

Composition with Pipe Operators

All factories work directly in pipelines:

cpp
// Corresponding test: test_then_upon.cpp
// Pipeline chain
auto pipeline = just(1)
              | then([](int x) { return x + 1; })
              | upon_error([](auto e) { return -1; })
              | upon_stopped([] { return 0; });

// Or dynamically returned in let_value
auto result = sync_wait(
  schedule(sch)
    | let_value([] {
        if (rand() % 2 == 0)
          return just(42);
        else
          return just_error(std::make_exception_ptr(std::runtime_error{"bad luck"}));
      })
);

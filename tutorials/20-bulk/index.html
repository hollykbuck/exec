20 · bulk — Data Parallel Operations

Standard section: [exec.bulk] stdexec reference: examples/nvexec/bulk.cu Related tests: tests/test_bulk.cpp

Overview

bulk applies an operation in parallel over a set of indices. It is a sender adaptor for data-level parallelism.

cpp
namespace std::execution {
  inline constexpr bulk_t bulk{};
  inline constexpr bulk_chunked_t bulk_chunked{};
  inline constexpr bulk_unchunked_t bulk_unchunked{};
}

Basic Syntax

cpp
bulk(sndr, shape, f)
// shape: size of the execution domain
// f: invocable(Idx, Ts...) → void
// For each idx ∈ [0, shape), call f in parallel
cpp
// Corresponding test: test_bulk.cpp → "bulk applies operation over index range"
auto result = sync_wait(
  just(100)
    | bulk(10, [](int idx, int& total) {
        // idx ∈ [0, 10), total = 100
        // Can execute in parallel on different threads
        printf("idx %d, total %d\n", idx, total);
      })
);

Execution Semantics of bulk

graph LR
  A["Input: set_value(Ts...)"] --> B["For each idx in shape"]
  B --> C["Execute f(idx, Ts...)<br/>can be parallel on<br/>different execution agents"]
  C --> D["Output: set_value(Ts...)<br/>passes through<br/>the previous sender's values"]

Key Properties

  • shape defines the parallel index space [0, shape)
  • f is called once per index, calls may be on different execution agents
  • f receives the index and the previous sender’s result values (as references)
  • Side effects of f (writing shared data) require external synchronization

Parallel Sum Example

cpp
// Corresponding test: test_bulk.cpp → "bulk with state from upstream"
std::vector<int> data = {1, 2, 3, 4, 5, 6, 7, 8};
std::atomic<int> sum{0};

sync_wait(
  just(std::ref(data), std::ref(sum))
    | bulk(data.size(), [](int idx, auto& data, auto& sum) {
        sum.fetch_add(data[idx], std::memory_order_relaxed);
      })
);

// sum = 36

bulk_chunked and bulk_unchunked

The standard provides three versions of bulk:

VersionBehavior
bulkDefault implementation, may or may not chunk
bulk_chunkedGuarantees chunking (divides range into contiguous chunks, each chunk executes on one execution agent)
bulk_unchunkedGuarantees no chunking (one execution agent per index)

Bulk on GPU

nvexec (NVIDIA’s GPU backend) provides a GPU implementation of bulk:

cpp
// Corresponding test: (no direct test)
nvexec::stream_scheduler gpu_sch{};

// Process 1024 elements in parallel on GPU
starts_on(gpu_sch,
  just(device_buffer)
    | nvexec::bulk(1024, [] __device__(int idx, device_buffer& buf) {
        buf[idx] = buf[idx] * 2 + 1;
      })
);

From examples/nvexec/bulk.cu: bulk operations on GPU are implemented by CUDA kernels, with each thread processing one index.

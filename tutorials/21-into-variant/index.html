Overview

These three sender adaptors handle sender type unification and signal conversion:

AdaptorPurpose
into_variant(sndr)Unifies all possible value types into variant<Ts...>
stopped_as_optional(sndr)Converts set_stopped to set_value(optional<T>)
stopped_as_error(sndr, err)Converts set_stopped to set_error

into_variant — Unifying Value Types

When a sender has multiple possible value completion signatures, into_variant unifies them into a single variant:

cpp
// Corresponding test: test_into_variant.cpp → "into_variant wraps value in variant of tuple"
// Conditional branches produce different value types
sender auto branch(int x) -> sender auto {
  if (x > 0)
    return just(x);                // set_value_t(int)
  else
    return just();                 // set_value_t()
}

// Direct use: two different value types
auto s1 = just(42) | then([](int x) { return branch(x); }) | sync_wait;
// Error: value_types_of_t has multiple types

// Unify with into_variant
auto s2 = just(42)
        | then([](int x) { return branch(x); })
        | into_variant()
        | sync_wait;
// Result: variant<tuple<int>, tuple<>>

stopped_as_optional — Converting Cancellation to a Value

cpp
template<sender Sndr>
sender auto stopped_as_optional(Sndr sndr);

If the sender is cancelled (set_stopped), convert it to set_value(std::optional<T>{}):

cpp
// Corresponding test: test_into_variant.cpp → "stopped_as_optional preserves set_value"
auto result = sync_wait(
  just(42)
    | stopped_as_optional
);
// result = optional{tuple{optional{42}}}

auto cancelled = sync_wait(
  just_stopped()
    | stopped_as_optional
);
// result = optional{tuple{optional<int>{}}}  (value = nullopt)

Usage in Coroutines

cpp
// Corresponding test: test_coroutine.cpp → "stopped_as_optional in coroutine"
sender auto async_fn() -> stdexec::task<std::optional<int>> {
  co_return co_await stopped_as_optional(some_sender);
}
// If some_sender is cancelled, async_fn returns nullopt

stopped_as_error — Converting Cancellation to an Error

cpp
template<sender Sndr, class Error>
sender auto stopped_as_error(Sndr sndr, Error err);
cpp
// Corresponding test: test_into_variant.cpp
auto result = sync_wait(
  just_stopped()
    | stopped_as_error(std::make_exception_ptr(
        std::runtime_error{"operation cancelled"}))
);
// throws runtime_error

Use Cases for Type Erasure

into_variant is particularly useful when you need to store heterogeneous sender types in a container:

cpp
// Corresponding test: test_into_variant.cpp
// Storing in a vector requires a unified type
std::vector<sender auto> senders;  // Error: heterogeneous types

// Use into_variant to unify types
std::vector<decltype(just(0) | into_variant())> senders;
senders.push_back(just(42) | into_variant());
senders.push_back(just() | into_variant());
// OK: all elements have the same type

Relationship with when_all_with_variant

when_all_with_variant is essentially the combination of when_all + into_variant:

cpp
// Equivalent (approximately)
when_all(s1 | into_variant(), s2 | into_variant())

The difference is that when_all_with_variant handles variant expansion internally within when_all, which may be more efficient.

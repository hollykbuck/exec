27 · tag_invoke & CPO

Standard Section: [exec.general] stdexec Reference: include/stdexec/functional.hpp Related Tests: tests/test_cpo.cpp

Overview

tag_invoke is the customization point mechanism of std::execution. It allows users to extend the library’s behavior via ADL lookup.

A CPO (Customization Point Object) is a callable object that uses tag_invoke. When a CPO is invoked, it attempts to:

  1. If the argument type has a member function with the same name, call it
  2. Otherwise, perform ADL lookup via tag_invoke

Basic Usage of tag_invoke

cpp
// Corresponding test: test_cpo.cpp → "custom CPO via member function is callable"
// Define CPO type
struct my_cpo_t {
  // Default behavior of the CPO
  template<class T>
  auto operator()(T&& t) const
    -> decltype(tag_invoke(*this, std::forward<T>(t)))
  {
    return tag_invoke(*this, std::forward<T>(t));
  }
};
inline constexpr my_cpo_t my_cpo{};

// Specialize for user type
struct my_type {
  friend int tag_invoke(my_cpo_t, const my_type&) {
    return 42;
  }
};

my_cpo(my_type{});  // → 42 (found tag_invoke via ADL)

CPOs in stdexec

All core operations in stdexec are CPOs:

cpp
// schedule CPO
inline constexpr schedule_t schedule{};
// invoke schedule(sch)
// → tag_invoke(schedule_t{}, sch)
// → if sch has a member schedule(), call it

// connect CPO
inline constexpr connect_t connect{};
// invoke connect(snd, rcvr)
// → tag_invoke(connect_t{}, snd, rcvr)
// → if snd has a member connect(rcvr), call it

Custom Scheduler (via tag_invoke)

cpp
// Corresponding test: test_cpo.cpp → "custom scheduler with member schedule() is callable"
struct my_scheduler {
  // Implement get_forward_progress_guarantee via tag_invoke
  friend auto tag_invoke(
    stdexec::tag_t<stdexec::get_forward_progress_guarantee>,
    const my_scheduler&) noexcept
    -> stdexec::forward_progress_guarantee
  {
    return stdexec::forward_progress_guarantee::concurrent;
  }

  bool operator==(const my_scheduler&) const = default;
};

// Implement schedule for my_scheduler
struct my_schedule_sender { /* ... */ };

my_schedule_sender tag_invoke(
  stdexec::tag_t<stdexec::schedule>,
  const my_scheduler& sch)
{
  return {};
}

tag_t Helper

cpp
// tag_t<CPO> returns the type of the CPO

CPO Implementation Pattern in stdexec

CPOs in stdexec mostly follow this pattern:

cpp
// 1. Define CPO type (empty struct)
struct schedule_t {
  // 2. Define operator(), calling tag_invoke
  template<scheduler Sch>
  auto operator()(Sch&& sch) const
    noexcept(noexcept(tag_invoke(*this, std::forward<Sch>(sch))))
    -> decltype(tag_invoke(*this, std::forward<Sch>(sch)))
  {
    return tag_invoke(*this, std::forward<Sch>(sch));
  }
};

// 3. Define inline constexpr variable
inline constexpr schedule_t schedule{};

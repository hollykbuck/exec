28 · Stop Token & Cancellation

Standard Section: [exec.get.stop.token] stdexec Reference: include/stdexec/stop_token.hpp Related Tests: tests/test_stop_token.cpp

Overview

The stop token is the cooperative cancellation mechanism of std::execution.

cpp
namespace std::execution {
  struct get_stop_token_t {};
  inline constexpr get_stop_token_t get_stop_token{};

  template<class T>
    using stop_token_of_t = remove_cvref_t<decltype(get_stop_token(declval<T>()))>;
}

Core Types

TypeBehavior
inplace_stop_tokenA token that can query stop state
inplace_stop_sourceA source that can request a stop
never_stop_tokenA token that never stops (compile-time optimization)

inplace_stop_source

cpp
// Corresponding test: test_stop_token.cpp → "inplace_stop_source requests stop"
stdexec::inplace_stop_source source;

// Get a token
stdexec::inplace_stop_token token = source.get_token();

// Request stop
source.request_stop();

// Query
bool stopped = token.stop_requested();  // true

never_stop_token

cpp
// Corresponding test: test_stop_token.cpp → "never_stop_token never stops"
stdexec::never_stop_token token;

// Known at compile time: never stops
bool stopped = token.stop_requested();  // false (constant)

Using in a Pipeline

Getting the Stop Token from the Environment

cpp
// Corresponding test: test_stop_token.cpp → "stop_requested via read_env"
// Get the stop token inside a sender
sender auto check_stop() {
  return read_env(get_stop_token)
       | then([](auto token) {
           if (token.stop_requested()) {
             // Cooperative cancellation checkpoint
           }
         });
}

Creating a Cancellable Pipeline

cpp
stdexec::inplace_stop_source stop_source;

// Provide a stop token in the receiver's environment
struct cancellable_receiver {
  stdexec::inplace_stop_token token_;

  auto get_env() const noexcept {
    return stdexec::prop{get_stop_token, token_};
  }
  // ... set_value, set_error, set_stopped
};

// Request stop from another thread
// stop_source.request_stop();

Cancellation Propagation

Cancellation propagates through the receiver’s environment:

graph TD
  A["sender"] --> B["connect(rcvr)"]
  B --> C["op_state"]
  C --> D["get_env(rcvr)"]
  D --> E["Environment { get_stop_token: token }"]
  E --> F["Sender can check internally:"]
  F --> G["if (token.stop_requested())"]
  G --> H["→ set_stopped(...)"]

Key Cancellation Points

Sender algorithms should check the stop token at:

  • Checkpoints inside long-running loops
  • Before starting child operations
  • Before waiting on external resources
cpp
void start() & noexcept {
  if (get_stop_token(get_env(rcvr_)).stop_requested()) {
    set_stopped(std::move(rcvr_));
    return;
  }
  // Continue execution...
}

unstoppable and write_env

Cancellation can be disabled:

cpp
// Disable cancellation
sender auto work = cancellable_op
                 | unstoppable
                 | then([](auto x) {
                     // No longer cancellable here
                     return x;
                   });

Cooperation with async_scope

async_scope requests cancellation of all outstanding operations upon destruction:

cpp
{
  exec::async_scope scope;
  scope.spawn(long_running_work);
  // scope destructor → request all child operations to stop
}  // Waits for all operations to complete (via cancellation)

Stop Token Implementation Patterns

Checking Stop

cpp
if (auto token = get_stop_token(get_env(rcvr_)); token.stop_requested()) {
  set_stopped(std::move(rcvr_));
  return;
}

Registering a Stop Callback

cpp
struct callback {
  stdexec::inplace_stop_token token_;
  operation_state* op_;

  void operator()() noexcept {
    // Stop requested → perform cleanup
    op_->cancel();
  }
};

// Register callback
auto cb = token_.stop_callback(callback{token_, this});
// When stop_source.request_stop() is called, callback() is executed

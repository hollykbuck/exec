29 · Structured Concurrency — async_scope / counting_scope

Standard section: [exec.scope] stdexec reference: exec/async_scope.hpp, examples/scope.cpp Related tests: tests/test_spawn_scope.cpp

Overview

Structured concurrency is a key design goal of std::execution: guaranteeing that all child operations complete before the parent operation completes, preventing task leaks and resource leaks.

cpp
namespace std::execution {
  template<class Token>
    concept scope_token = /* ... */;

  template<class Token>
    concept scope_association = /* ... */;

  class simple_counting_scope;
  class counting_scope;
}

Why Structured Concurrency?

Without structured concurrency:

cpp
void bad() {
  spawn(async_work());  // fire-and-forget
  // The function returns, but async_work may still be executing!
  // Local variables referenced by async_work may have been destroyed
}

With structured concurrency:

cpp
void good() {
  exec::async_scope scope;
  scope.spawn(async_work());
  // Before the function exits, wait for all child operations to complete
  sync_wait(scope.on_empty());
  // Now it is safe to destroy local variables
}

async_scope (stdexec extension)

exec::async_scope provides structured concurrency features beyond the p2300 standard:

cpp
namespace exec {
  class async_scope {
  public:
    // Fire-and-forget
    void spawn(sender auto sndr);

    // Spawn and return a sender that can observe the result
    sender auto spawn_future(sender auto sndr);

    // Bind the sender's lifetime to the scope
    sender auto nest(sender auto sndr);

    // Wait for the scope to be empty
    sender auto on_empty() noexcept;

    // Request all child operations to stop
    void request_stop() noexcept;
  };
}

spawn — Fire-and-Forget

cpp
exec::async_scope scope;

// Spawn work in the scope
scope.spawn(schedule(sch) | then(work1));
scope.spawn(schedule(sch) | then(work2));

// Wait for all work to complete
sync_wait(scope.on_empty());

spawn_future — Spawn and Observe Results

cpp
// spawn_future returns a sender that can observe the result
sender auto future = scope.spawn_future(
  schedule(sch) | then([] { return 42; })
);

// Wait for the result later
auto [result] = sync_wait(std::move(future)).value();

nest — Bind Lifetime

cpp
sender auto nested = scope.nest(schedule(sch) | then(work));

// nested can be passed around outside the scope
// nested will only complete when the scope is empty

Complete Example

From examples/scope.cpp:

cpp
exec::static_thread_pool ctx{1};
exec::async_scope scope;

auto sch = ctx.get_scheduler();

// spawn
scope.spawn(schedule(sch) | then([] { printf("void\n"); }));

// spawn_future
sender auto fortyTwoFuture = scope.spawn_future(
  schedule(sch) | then([] { return 42; })
);

// Wait for all to complete
sync_wait(when_all(
  scope.on_empty(),
  fortyTwoFuture | then([](int x) { printf("%d\n", x); })
));

// nest
{
  sender auto nest = scope.nest(schedule(sch));
  // nest is valid within the scope's lifetime
}
sync_wait(scope.on_empty());

Structured Concurrency Guarantees

graph TD
  A["scope created"]
  A --> B["spawn / spawn_future / nest adds work"]
  B --> C["work begins executing"]
  C --> D["scope destructor"]
  D --> E["1. Request all outstanding work to stop"]
  E --> F["2. Wait for all work to complete (or stop)"]
  F --> G["3. Destroy scope"]

This means: a scope object is never destroyed before its child operations complete.

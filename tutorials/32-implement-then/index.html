32 · Implementing then from Scratch — Custom Sender + Receiver + Operation State

Standard section: [exec.then] stdexec reference: examples/algorithms/then.hpp, examples/then.cpp Related tests: tests/test_impl_then.cpp

Overview

Implementing then from scratch is the most effective way to understand the P2300 sender/receiver model. This section will build a complete then algorithm step by step.

Step 1: Define the Then Sender

cpp
// Corresponding test: test_impl_then.cpp → "impl then: transforms int values"
template <stdexec::sender Sndr, std::invocable<std::decay_t<...>> Fn>
struct then_sender {
  using sender_concept = stdexec::sender_tag;

  Sndr sndr_;
  Fn fn_;

  explicit then_sender(Sndr sndr, Fn fn)
    : sndr_(std::move(sndr))
    , fn_(std::move(fn))
  {}
};

Step 2: Declare Completion Signatures

Completion signatures may depend on the sender’s value category and environment, so we use the get_completion_signatures member function template:

cpp
template <class... Args>
using _set_value_t =
  stdexec::completion_signatures<stdexec::set_value_t(std::invoke_result_t<F, Args...>)>;

template <class... Env>
using _completions_t = stdexec::__transform_completion_signatures_t<
  stdexec::completion_signatures_of_t<Sndr, Env...>,
  stdexec::completion_signatures<stdexec::set_error_t(std::exception_ptr)>,
  _set_value_t>;

// Inside then_sender:
template <class Self, class... Env>
static consteval auto get_completion_signatures() -> _completions_t<Env...> {
  return {};
}

Step 3: Define the Operation State

cpp
template <stdexec::receiver R, class Sndr, class Fn>
struct then_op {
  // Input sender's operation state type
  using op_t = stdexec::connect_result_t<Sndr&, /* internal receiver */>;

  Sndr sndr_;
  Fn fn_;
  R rcvr_;
  op_t op_;

  then_op(Sndr sndr, Fn fn, R rcvr)
    : sndr_(std::move(sndr))
    , fn_(std::move(fn))
    , rcvr_(std::move(rcvr))
    , op_(connect())  // Connect internal receiver
  {}

  void start() & noexcept {
    stdexec::start(op_);
  }

private:
  auto connect() {
    return stdexec::connect(sndr_, /* internal receiver */);
  }
};

Step 4: Define the Internal Receiver

The internal receiver receives the input sender’s completion signals, transforms them, and forwards them to the external receiver:

cpp
// Corresponding test: test_impl_then.cpp → "impl then: void function"
template <class Op>
struct then_receiver {
  using receiver_concept = stdexec::receiver_tag;

  Op* op_;

  // Received set_value → call fn, forward result
  template <class... Args>
  void set_value(Args&&... args) && noexcept {
    if constexpr (std::is_nothrow_invocable_v<Fn, Args...>) {
      // Non-throwing → forward directly
      if constexpr (std::same_as<void, std::invoke_result_t<Fn, Args...>>) {
        std::invoke(op_->fn_, std::forward<Args>(args)...);
        stdexec::set_value(std::move(op_->rcvr_));
      } else {
        stdexec::set_value(
          std::move(op_->rcvr_),
          std::invoke(op_->fn_, std::forward<Args>(args)...));
      }
    } else {
      // May throw → try-catch
      STDEXEC_TRY {
        if constexpr (std::same_as<void, std::invoke_result_t<Fn, Args...>>) {
          std::invoke(op_->fn_, std::forward<Args>(args)...);
          stdexec::set_value(std::move(op_->rcvr_));
        } else {
          stdexec::set_value(
            std::move(op_->rcvr_),
            std::invoke(op_->fn_, std::forward<Args>(args)...));
        }
      }
      STDEXEC_CATCH_ALL {
        stdexec::set_error(std::move(op_->rcvr_), std::current_exception());
      }
    }
  }

  // Forward errors
  void set_error(auto&& e) && noexcept {
    stdexec::set_error(std::move(op_->rcvr_), std::forward<decltype(e)>(e));
  }

  // Forward cancellation
  void set_stopped() && noexcept {
    stdexec::set_stopped(std::move(op_->rcvr_));
  }

  // Forward environment
  auto get_env() const noexcept {
    return stdexec::get_env(op_->rcvr_);
  }
};

Step 5: Implement connect

cpp
template <class Sndr, class Fn>
struct then_sender {
  // ...

  template <stdexec::receiver R>
    requires stdexec::sender_to<Sndr, then_receiver<then_op<Sndr, Fn, R>>>
  auto connect(R r) && -> then_op<Sndr, Fn, R> {
    return then_op<Sndr, Fn, R>{
      std::move(sndr_),
      std::move(fn_),
      std::move(r)
    };
  }
};

Step 6: Completion Signature Transformation

Value signature transformation: convert set_value_t(Args...) to set_value_t(std::invoke_result_t<Fn, Args...>):

cpp
template <class... Args>
using _set_value_t =
  stdexec::completion_signatures<stdexec::set_value_t(std::invoke_result_t<Fn, Args...>)>;

Use __transform_completion_signatures_t to transform the input sender’s completion signatures as a whole, and add a default set_error_t(std::exception_ptr):

cpp
template <class... Env>
using _completions_t = stdexec::__transform_completion_signatures_t<
  stdexec::completion_signatures_of_t<Sndr, Env...>,
  stdexec::completion_signatures<stdexec::set_error_t(std::exception_ptr)>,
  _set_value_t>;

Error and stopped signatures are forwarded by default (when no error transformation template argument is specified, __transform_completion_signatures_t retains the original error signatures).

Step 7: Adaptor Closure

cpp
struct then_adaptor {
  Fn fn_;

  template <stdexec::sender Sndr>
  auto operator()(Sndr sndr) {
    return then_sender<Sndr, Fn>{std::move(sndr), std::move(fn_)};
  }
};

Complete Implementation Summary

The core elements involved in implementing then from scratch:

  1. Sender type: holds the child sender and the function
  2. Completion signatures: compile-time computation of transformed signatures
  3. Operation state: holds the child operation state, function, and external receiver
  4. Internal receiver: receives the child operation’s result, executes the function, forwards to the external receiver
  5. connect(): creates the operation state
  6. start(): starts the child operation
  7. Pipe adaptor: supports operator|

stdexec’s examples/algorithms/then.hpp is a complete reference implementation containing all of the above elements.

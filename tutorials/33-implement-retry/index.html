33 · Implementing retry from Scratch — upon_error Recovery Pattern

Standard section: — stdexec reference: examples/algorithms/retry.hpp, examples/retry.cpp Related tests: tests/test_impl_retry.cpp

Overview

retry is a sender algorithm that automatically reconnects on failure. Its core mechanism is the upon_error recovery pattern — catching the set_error signal and re-initiating the operation.

Step 1: A Sender That Sometimes Fails

First, define a sender that fails on the first N invocations and succeeds afterward:

cpp
// Corresponding test: test_impl_retry.cpp → "fail_some eventually succeeds"
struct fail_some {
  using sender_concept = ex::sender_tag;
  using completion_signatures = ex::completion_signatures<
    ex::set_value_t(int),
    ex::set_error_t(std::exception_ptr)
  >;

  mutable int failure_count_ = 2;

  template <class R>
  struct op {
    R r_;
    const fail_some* parent_;
    void start() & noexcept {
      if (parent_->failure_count_ > 0) {
        --parent_->failure_count_;
        ex::set_error(std::move(r_), std::exception_ptr{});
      } else {
        ex::set_value(std::move(r_), 42);
      }
    }
  };

  template <class R>
  auto connect(R r) const -> op<R> {
    return op<R>{std::move(r), this};
  }
};

Step 2: Manual Retry Based on upon_error

upon_error converts set_error to set_value, which is the foundation for implementing retry:

cpp
// Corresponding test: test_impl_retry.cpp → "upon_error recovers from failures"
bool recovered = false;

auto [v] = ex::sync_wait(
  ex::just_error(std::make_exception_ptr(std::runtime_error{"fail"}))
    | ex::upon_error([&](std::exception_ptr) { recovered = true; return 42; })
).value();

// recovered == true, v == 42

Step 3: Retry Loop via let_value

Combining let_value allows re-initiating the operation after recovery:

cpp
auto try_with_retry = [](int max_attempts) -> ex::sender auto {
  return ex::let_value(ex::just(0), [max_attempts](int) -> ex::sender auto {
    // Here we can reconnect the original sender and retry
    return ex::just(42);
  });
};

auto [v] = ex::sync_wait(try_with_retry(3)).value();
// v == 42

Complete retry Implementation Pattern

stdexec’s actual retry implementation (see examples/algorithms/retry.hpp) uses two key patterns:

1. receiver_adaptor — Intercept Only set_error

cpp
struct _retry_receiver : exec::receiver_adaptor<_retry_receiver> {
  _op* o_;

  auto base() && noexcept -> auto& { return o_->r_; }

  // Only intercept set_error; other signals are forwarded via receiver_adaptor to base
  void set_error(auto&&) && noexcept {
    o_->_retry();  // Error → retry!
  }
};

receiver_adaptor automatically forwards un-overridden set_value and set_stopped to the base() receiver.

2. std::optional + emplace to Rebuild Operation State

cpp
struct _op {
  // Core: child operation state stored in optional
  std::optional<connect_result_t<S&, _retry_receiver>> o_;

  void _retry() noexcept {
    STDEXEC_TRY {
      o_.emplace(connect(s_, _retry_receiver{this}));  // Rebuild!
      start(*o_);                                       // Restart
    }
    STDEXEC_CATCH_ALL {
      set_error(static_cast<R&&>(r_), std::current_exception());
    }
  }
};

Completion Signature Transformation

retry swallows set_error, so its completion signature has no error signature:

cpp
template <class Self, class... Env>
static consteval auto get_completion_signatures()
  -> ex::__transform_completion_signatures_t<
    ex::completion_signatures_of_t<S&, Env...>,
    ex::completion_signatures<ex::set_error_t(std::exception_ptr)>,
    _value,
    _error>      // _error maps to empty → remove all error signatures
{
  return {};
}

Key Takeaways

  1. upon_error is the foundation of retry — it recovers errors into the normal value path
  2. std::optional + emplace — allows rebuilding the child operation within the operation state’s lifetime
  3. receiver_adaptor — conveniently intercept only the signal of interest, forward the rest
  4. Infinite retry vs bounded retry — the above pattern implements infinite retry; adding a counter gives bounded retry
cpp
// Bounded retry: combine with a counter
struct retry_with_limit {
  int max_retries_;
  // In _retry(), check the counter; exceed limit then forward set_error
};

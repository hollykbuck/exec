36 · Linux io_uring

Standard Section: — stdexec Reference: exec/linux/io_uring_context.hpp, examples/io_uring.cpp

Overview

io_uring is a high-performance asynchronous I/O framework available since Linux 5.1. stdexec provides io_uring_context as a scheduler, integrating async file I/O into the sender pipeline.

cpp
namespace exec {
  class io_uring_context {
  public:
    class scheduler;

    scheduler get_scheduler() noexcept;

    void run();       // Event loop
    void stop();      // Request stop
    void reset();     // Reset
  };
}

Basic Usage

cpp
exec::io_uring_context ctx;

// Get scheduler
auto sched = ctx.get_scheduler();

// Schedule a timer on io_uring
sync_wait(
  schedule(sched)
    | then([] { std::cout << "from io_uring\n"; })
);

// Or drive on a separate thread
std::thread t([&] { ctx.run(); });

io_uring Example

From examples/io_uring.cpp:

cpp
exec::io_uring_context io_ctx;
auto io_sched = io_ctx.get_scheduler();

// Schedule timer
sync_wait(
  schedule_at(io_sched, 100ms)
    | then([] { printf("timer fired\n"); })
);

// Wait for multiple io_uring operations
sync_wait(when_all(
  schedule_at(io_sched, 50ms)  | then(t1),
  schedule_at(io_sched, 100ms) | then(t2),
  schedule_at(io_sched, 150ms) | then(t3)
));

// finally guarantees cleanup
sync_wait(
  schedule_at(io_sched, 100ms)
    | finally(schedule(io_sched) | then([] {
        printf("cleanup\n");
      }))
);

Supported io_uring Operations

OperationFunction
Schedule immediatelyschedule(sched)
Schedule at (duration)schedule_at(sched, duration)
Schedule at (absolute time)schedule_at(sched, time_point)
TimeoutVia when_any composition

Combining with Thread Pool

cpp
exec::io_uring_context io_ctx;
exec::static_thread_pool cpu_pool{4};

// I/O on io_uring, computation on thread pool
starts_on(io_ctx.get_scheduler(),
  then(read_file("data.bin"))
    | continues_on(cpu_pool.get_scheduler())
    | then([](buffer buf) { return process(buf); })
);

Lifecycle

cpp
exec::io_uring_context ctx;
auto sched = ctx.get_scheduler();

std::thread worker([&] { ctx.run(); });

// Submit work
spawn(schedule(sched) | then(work));

// Stop event loop
ctx.stop();
worker.join();
ctx.reset();  // Reusable

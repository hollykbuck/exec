37 · Windows Thread Pool

Standard Section: — stdexec Reference: exec/windows/windows_thread_pool.hpp Related Tests: tests/test_windows_thread_pool.cpp

Overview

stdexec provides a Windows Thread Pool backend, wrapping the Windows Thread Pool API as a scheduler.

cpp
namespace exec {
  class windows_thread_pool;
}

Basic Usage

cpp
// Corresponding test: test_windows_thread_pool.cpp → "windows_thread_pool executes work"
exec::windows_thread_pool pool;
auto sch = pool.get_scheduler();

sync_wait(
  schedule(sch)
    | then([] { printf("Executing on Windows thread pool\n"); })
);

Cross-Platform Code

The Windows thread pool is part of stdexec’s cross-platform strategy:

cpp
#if defined(_WIN32)
  using default_thread_pool = exec::windows_thread_pool;
#else
  using default_thread_pool = exec::static_thread_pool;
#endif

default_thread_pool pool{4};
auto sch = pool.get_scheduler();

Integration with I/O Completion Ports

The Windows thread pool scheduler can integrate I/O Completion Ports for async file I/O:

cpp
// Conceptual usage (specific API depends on encapsulation level)
exec::windows_thread_pool pool;
auto sched = pool.get_scheduler();

starts_on(sched,
  then([] {
    // Perform I/O on Windows thread pool
    return read_file_async("data.bin");
  })
);

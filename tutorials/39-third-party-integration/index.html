39 · Third-party Framework Integration

On this page

Standard Section: — stdexec Reference: execpools/, examples/benchmark/

Overview

stdexec provides adapters for several third-party frameworks, allowing you to use schedulers from ASIO, TBB, Taskflow, etc. as std::execution schedulers.

ASIO

cpp
#include <asioexec/as_default_on.hpp>
#include <asioexec/completion_token.hpp>
#include <asioexec/executor_with_default.hpp>

// ASIO io_context as execution context
asio::io_context io_ctx;
auto sch = /* obtain stdexec scheduler from io_context */;

starts_on(sch,
  then([] { return async_work(); })
  | continues_on(exec::static_thread_pool{}.get_scheduler())
  | then([](auto result) { process(result); })
);

TBB (Intel oneTBB)

cpp
#include <execpools/tbb/tbb_thread_pool.hpp>

execpools::tbb_thread_pool pool;
auto sch = pool.get_scheduler();

// Leverage TBB's work-stealing thread pool
starts_on(sch,
  bulk(1000, [](int idx) { compute(idx); })
);

Taskflow

cpp
#include <execpools/taskflow/taskflow_thread_pool.hpp>

execpools::taskflow_thread_pool pool;
auto sch = pool.get_scheduler();

// Leverage Taskflow's DAG task graph
starts_on(sch,
  when_all(
    task_a(),
    task_b(),
    task_c()
  )
);

Performance Benchmarks

examples/benchmark/ contains performance comparisons across multiple backends:

cpp
// static_thread_pool benchmark
benchmark::run<exec::static_thread_pool>();

// TBB benchmark
benchmark::run<execpools::tbb_thread_pool>();

// Taskflow benchmark
benchmark::run<execpools::taskflow_thread_pool>();

// ASIO benchmark
benchmark::run<asioexec::asio_thread_pool>();

Backend Selection Guide

BackendStrengthUse Case
static_thread_poolZero dependency, simple and reliableGeneral async work
TBBWork-stealing, nested parallelismCPU-intensive parallel computation
TaskflowDAG task graphParallel tasks with complex dependencies
ASIONetwork I/O integrationNetwork servers, existing ASIO code
nvexecGPU accelerationGPU computing

Extension Interface

To integrate your own framework, you need to implement:

  1. Scheduler concept (schedule, get_forward_progress_guarantee)
  2. Schedule sender (connect → operation state → start)
  3. Execution context (manage thread/resource lifecycle)

stdexec’s execpools/thread_pool_base.hpp provides a reference base class.
